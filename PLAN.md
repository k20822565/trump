# 엑셀 위장 카드게임 - 전체 계획서

## 프로젝트 개요
회사에서 몰래 즐길 수 있는 온라인 카드게임. 외관은 Microsoft Excel로 위장.  
실시간 멀티플레이 지원 (2~6명), 원카드·텍사스 홀덤·섯다 3종 제공.

---

## 기술 스택

| 구분 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | HTML + CSS + Vanilla JS | 단순, 빠름 |
| 백엔드 | Node.js + Express + Socket.io | 실시간 통신 최적, 설정 최소 |
| DB | 없음 (서버 메모리) | 게임 특성상 영속성 불필요 |
| 배포 | Render.com (무료 티어) | 무료, 슬립 허용 |

## UI/UX 컨셉 — 엑셀 위장

### 기본 레이아웃
- 전체 화면이 Excel 2016~2019 스타일
- 상단: Excel 제목바 ("통합 문서1 - Microsoft Excel") + 리본 메뉴 (홈/삽입/페이지레이아웃/...)
- 좌측: 행 번호 (1, 2, 3...)
- 상단: 열 헤더 (A, B, C, D...)
- 배경: 회색 격자선이 있는 실제 스프레드시트처럼 보이는 셀
- 배경 셀에는 그럴듯한 가짜 업무 데이터 (판매 실적 표, 숫자 등) 표시

### 게임 영역
- 게임 UI는 스프레드시트 셀 범위를 선택한 것처럼 보이는 영역에 렌더링
- 선택 영역 테두리 (Excel 파란/초록 선택 효과) 안에 게임 표시
- 카드, 버튼 등을 셀 스타일로 디자인

### 긴급 숨김 (ESC 키)
- ESC 누르면 게임 영역이 사라지고 가짜 업무 스프레드시트만 표시
- 다시 ESC 또는 특정 셀 클릭 시 게임 복귀
- 전환 시 자연스럽게 (애니메이션 없이 즉시)

## 게임 플로우

```
접속
  └→ 닉네임 입력 (로컬 저장)
       └→ 로비 (방 목록 표시)
            ├→ 방 생성 (방 이름, 게임 종류 선택)
            └→ 방 입장
                 └→ 게임 방
                      ├→ 대기 (호스트: 게임 종류 변경 / 시작 버튼)
                      └→ 게임 진행
                           ├→ 원카드: 승패 기록
                           ├→ 텍사스 홀덤: 칩 배팅
                           └→ 섯다: 칩 배팅
```

## 게임 규칙 상세

### 원카드
- 표준 52장 덱
- 특수 카드: +2 (누적 가능), 리버스, 스킵, 와일드(색 변경)
- 승패 기록만 표시 (칩 없음)
- 2~6명 플레이

### 텍사스 홀덤
- 표준 52장 덱
- 블라인드: 스몰 1칩, 빅 2칩
- 베팅 라운드: 프리플롭 → 플롭 → 턴 → 리버 → 쇼다운
- 핸드 랭킹: 로열플러시 ~ 하이카드
- 올인 지원

### 섯다
- 화투 40장 (1월~10월, 각 4장) 중 사용 카드 선별
- 2장씩 지급 후 베팅
- **패 랭킹 (높은 순)**:
  1. 장땡 (10+10)
  2. 9땡 ~ 1땡
  3. 알리 (1+2)
  4. 독사 (1+4)
  5. 구사 (9+4) — 특수패
  6. 장사 (10+4)
  7. 세륙 (3+6)
  8. 끗수 9끗 ~ 1끗 (합의 일의 자리)
  9. 망통 (0끗)
- **특수패**:
  - 암행어사 (3+8): 장땡 제외 모든 땡 이김
  - 구사 (9+4): 즉시 공개 시 판 무효 (재배팅) 또는 4끗으로 사용 선택
- 베팅: 콜/레이즈/다이

## 칩 시스템

- 시작 칩: 30개
- 원카드: 칩 사용 없음
- 텍사스 홀덤 / 섯다: 칩 배팅
- 파산 조건: 칩 0개
- **구걸 기능**: 파산 플레이어가 다른 특정 플레이어에게 칩 요청 → 상대방 수락/거절

## 방 목록 표시 항목

| 방 제목 | 게임 종류 | 참가 인원 | 상태 |
|---------|---------|---------|------|
| 홍길동의 방 | 텍사스 홀덤 | 2/6 | 대기중 |
| 점심시간 | 섯다 | 4/4 | 게임중 |

- 게임중인 방은 입장 불가 (관전 기능 없음)

## 배포 계획 (Render.com)

1. GitHub 저장소 생성 및 코드 업로드
2. Render.com 무료 계정 생성
3. New Web Service → GitHub 연결
4. Build Command: `npm install`
5. Start Command: `node server.js`
6. 무료 티어: 비활성 15분 후 슬립 (첫 요청 시 약 30초 웨이크업)
